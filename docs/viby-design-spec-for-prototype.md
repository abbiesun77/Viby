# Viby — 前端原型设计规格文档

> 供 Open Design 等原型工具使用。本文档描述每一个屏幕的布局、组件、交互状态和导航出口。

---

## 屏幕总地图

```
[S01] Landing Page
  ↓ 点击「免费试用」
[S02] Sign Up
  ↓ 提交注册
[S03] Email Verification（等待验证提示页）
  ↓ 用户点击邮件链接后
[S04] Onboarding — 入口选择
  ↓ 选择路径 A（想法）
[S05] Onboarding — 引导对话（想法路径）
  ↓ 三问完成，生成剧本
[S06] Tab 1：剧本 Preview & 编辑
  ↓ 确认剧本
  ↓ 选择路径 B（直接进入 S07）
[S07] Tab 2：场景 & 分镜
  ↓ 补充资产（可选）
[S08] 资产生成对话（覆层/右侧面板）
  ↓ 确认场景
[S09] Tab 3：Storyboard
  ↓ 所有场景 Storyboard 生成完毕
[S10] 导出
  ↓ 随时可从工作台进入
[S11] 项目工作台（Dashboard）
  ↓ 点击任意模块重入
  → S06 / S07 / S09 / S08 / S10
  ↓ 设置入口
[S12] 设置（BYOK）

[S13] Sign In（从 Landing 或注册页进入）
```

## S01 · Landing Page

### 目的
解释 Viby 是什么，驱动用户注册试用。

### 布局结构（从上到下）

**区域 1：顶部导航栏**
- 左：Logo「Viby」（衬线字体）
- 右：`登录`（文字链接）、`免费试用`（实心按钮）
- 行为：导航栏固定在顶部，滚动时保持可见

**区域 2：Hero 区**
- 标签行：`AI Video Pre-Production`（等宽小字）
- 主标题：`少抽卡，更稳地做出想要的视频`（大号衬线字体，逐字出现动画）
- 副标题：`把一句话或剧本整理成分镜、角色设定和场景参考图，打包成可直接交给视频模型的素材包。`
- 按钮组：
  - 主按钮：`免费试用 — 注册即得 Credit` → 跳转 S02
  - 文字链接：`了解工作流 →`（锚点滚动到区域 3）
- 统计数据行（三列，横线分隔）：
  - `2` / 种创作入口
  - `120` / 新用户 Viby Credit
  - `1` / 导出压缩包，直接用

**区域 3：工作流四步**（标题：「从想法到可投喂的素材包」）
- 四列网格，每列：
  - 大号数字（01/02/03/04，灰色衬线）
  - 步骤名称
  - 步骤说明
- 01：输入想法或剧本
- 02：生成剧本与场景分场
- 03：展开分镜卡与参考图
- 04：导出素材包

**区域 4：两种入口说明**
- 标题：「从你有的东西开始」
- 两行列表：
  - 行 1：`想法 / 故事梗概` · 说明文字 · 标签「需要扩写」
  - 行 2：`完整剧本` · 说明文字 · 标签「直接解析」

**区域 5：Trial Credit 区**
- 左：标题「先用，再决定要不要接自己的 Key」+ 说明
- 右：Credit 展示框（`120 Credit`，说明：新用户注册即得）+ 按钮组
  - 主按钮：`免费注册` → S02
  - 次按钮：`已有账号 — 登录` → S13

**区域 6：页脚**
- 左：Logo
- 右：版权文字

### 导航出口
| 触发 | 目标 |
|------|------|
| 「免费试用」按钮 | S02 Sign Up |
| 「登录」链接 | S13 Sign In |
| 「免费注册」按钮 | S02 Sign Up |
| 「已有账号 — 登录」| S13 Sign In |

## S02 · Sign Up

### 目的
用户注册，注册成功后触发邮件验证。

### 布局结构

- 顶部：Logo（可点击返回 S01）
- 卡片居中：
  - 标题：`创建 Viby 账号`
  - 副标题：`注册后立即获得 120 Viby Credit，先体验完整流程再决定是否接入自己的 Key。`
  - 字段：`邮箱`（文本输入）、`密码`（密码输入，至少 8 位）
  - 按钮：`创建账号`（提交）
  - 底部链接：`已有账号？登录 →` → S13

### 状态
- 默认：表单可编辑
- 加载中：按钮变为 `创建中...`，禁用输入
- 错误：字段下方红色提示文字（邮箱已注册 / 密码太短）
- 成功：跳转 S03

### 导航出口
| 触发 | 目标 |
|------|------|
| 提交成功 | S03 Email Verification |
| 「登录」链接 | S13 Sign In |
| Logo | S01 Landing |

## S13 · Sign In

### 布局结构

与 S02 镜像，差异：
- 标题：`登录 Viby`
- 无副标题
- 无 Credit 说明
- 按钮：`登录`
- 底部链接：`没有账号？免费注册 →` → S02

### 状态
- 错误：`邮箱或密码不正确`
- 成功：跳转 S11 项目工作台（如有已有项目）或 S04 Onboarding

### 导航出口
| 触发 | 目标 |
|------|------|
| 登录成功（首次） | S04 Onboarding |
| 登录成功（已有项目） | S11 工作台 |
| 「注册」链接 | S02 Sign Up |

## S03 · Email Verification

### 布局结构

- 居中提示卡：
  - 图标：信封
  - 标题：`请验证你的邮箱`
  - 说明：`我们已向 [用户邮箱] 发送了一封验证邮件，点击邮件中的链接后即可进入 Viby。`
  - 次要操作：`重新发送验证邮件`（文字链接，有冷却计时）
  - 底部：`用不同邮箱注册` → S02

### 状态
- 等待中：默认展示
- 重发冷却：按钮显示倒计时 `60s 后可重发`
- 验证成功（用户点击邮件链接后）：自动跳转 S04

## S04 · Onboarding — 入口选择

### 目的
新用户首次进入，选择创作路径。

### 布局结构

- 顶部：Logo + 右上角「Credit 余额：120」
- 中央卡片：
  - 标题：`你现在手上有什么？`
  - 两个大选项卡（卡片式，点击高亮选中）：

```
┌──────────────────────┐  ┌──────────────────────┐
│  💡 我有一个想法       │  │  📄 我已有完整剧本     │
│                       │  │                       │
│  一句话或故事梗概都    │  │  系统将直接解析你的    │
│  可以，系统会帮你      │  │  剧本内容，不会再      │
│  生成完整剧本。        │  │  重新生成剧本。        │
└──────────────────────┘  └──────────────────────┘
```

  - 输入框（选中后展开）：
    - 路径 A 选中：`输入你的想法或故事梗概...`（多行文本）
    - 路径 B 选中：`粘贴你的完整剧本...`（多行文本）+ 提示「⚠️ 选择此入口后系统不会再生成剧本，将直接进入场景解析」
  - 按钮：`开始` → 路径 A 进 S05，路径 B 进 S07（跳过剧本 Tab）

### 状态
- 默认：两卡片未选中，无输入框，按钮禁用
- 选中 + 有输入：按钮激活

### 导航出口
| 触发 | 目标 |
|------|------|
| 路径 A + 开始 | S05 引导对话 |
| 路径 B + 开始 | S07 Tab 2（跳过 S05/S06）|

## S05 · Onboarding — 引导对话（想法路径）

### 目的
三问引导用户确定风格/时长/情绪，生成剧本。

### 布局结构

进度行：`步骤 1/4：了解你的想法`

聊天气泡区（逐步出现）：

**问题 1**
```
Viby:  你想拍什么风格的视频？
[写实纪录感]  [电影感]  [广告感]  [动画/插画感]
```
用户点选 → 选项变成用户气泡，Viby 发下一问。

**问题 2**
```
Viby:  大概多长？
[15秒]  [30秒]  [60秒]  [自定义…]
```
选「自定义」展开输入框。

**问题 3**
```
Viby:  情绪基调呢？用形容词描述就行。
[___________________________] [发送]
```

三问完成 → Viby 气泡：`好，我来帮你生成剧本...` + 动画

生成完毕 → `剧本已生成，共 X 个场景。` + 按钮 `[查看并编辑剧本 →]`

### 状态
- 每问在上一问回答后才出现
- 生成中：loading 动画，禁止交互
- 生成失败：错误气泡 + `[重试]`

### 导航出口
| 触发 | 目标 |
|------|------|
| 「查看并编辑剧本」| S06 Tab 1 |

## S06 · Tab 1：剧本

### 布局结构

**顶部 Tab 栏**：`● 剧本  ○ 场景&分镜  ○ Storyboard`

**主内容（左右分栏）**：

```
== 场景 1：便利店夜间外景 ==（系统标注，灰色）

夜晚，便利店冷光招牌在街角亮着...（可直接编辑）

== 场景 2：室内走廊 ==
...
```

```

[历史对话]
```

**底部操作栏**：`[重新生成剧本]  [确认剧本，进入场景 →]`

### 状态
- AI 修改中：编辑器显示 loading 遮罩
- 确认后：Tab 2「场景&分镜」解锁

### 导航出口
| 触发 | 目标 |
|------|------|
| 「确认剧本，进入场景」| S07 Tab 2 |

## S07 · Tab 2：场景 & 分镜

### 布局结构

**顶部 Tab 栏**：`✓ 剧本  ● 场景&分镜  ○ Storyboard`

**场景列表（可折叠卡片）**：

```
▼ 场景 01 · 便利店夜间外景          [与 AI 调整本场景]
  ┌─────────────────────────────────────────────┐
  │ 镜 01  景别：远景  主体：便利店外景           │
  │        动作：镜头缓慢推进                   │
  │        氛围：孤寂、荧光蓝                   │
  │                                     [编辑]  │
  └─────────────────────────────────────────────┘
  [＋ 添加分镜]

▶ 场景 02（折叠）
```

**分镜卡编辑态**（点击「编辑」内联展开）：
```
景别/角度  [远景 ▼]
画面主体   [便利店外景        ]
动作/内容  [镜头缓慢向前推进  ]
情绪/氛围  [孤寂、荧光蓝冷色调]
                    [取消] [保存]
```

**资产缺口提示**（场景列表下方）：
```
┌───────────────────────────────────────────────────┐
│ 以下参考图缺失，建议在生成 Storyboard 前补充：     │
│                                                     │
│ 🔴 主角（男·30岁·航天服）    [生成三视图] [跳过]  │
│ 🔴 便利店室内空间             [生成空间图] [跳过]  │
│ 🟡 宇航头盔道具               [生成道具图] [跳过]  │
└───────────────────────────────────────────────────┘
```

**底部操作栏**：`[← 返回剧本]  [确认场景，生成 Storyboard →]`

### 状态
- 编辑中：分镜卡展开表单，其他卡片只读
- AI 调整中：该场景 loading
- 资产生成中：对应行 loading

| 触发 | 目标 |
|------|------|
| 资产「生成」| S08 资产生成面板（右抽屉）|
| 「确认场景，生成 Storyboard」| S09 Tab 3 |

## S08 · 资产生成面板（右抽屉）

### 目的
通过对话生成或替换单项资产。从 S07 和 S11 均可触发。

### 布局结构

右侧滑入抽屉（主内容区半透明，不全屏遮挡）：

```
资产：主角三视图                       [×]
─────────────────────────────────────
当前状态：未生成

Viby:  描述一下这个角色的外貌，越具体越好。

[                                   ]
[                              发送  ]
─────────────────────────────────────
预览

[生成后显示图片缩略图]

[重新生成]  [使用这张]  [上传替换]
```

### 状态
- 未生成：无预览，显示引导文字
- 生成中：loading 动画
- 已生成：图片缩略图 + 操作按钮
- 「使用这张」：面板关闭，对应缺口行变「✅ 已补充」

### 导航出口
| 触发 | 目标 |
|------|------|
| 「×」或「使用这张」| 返回来源页（S07 或 S11）|

## 通用组件规范

### Credit 角标
- 位置：所有工作区页面顶部右侧
- Credit < 20：橙色警告
- Credit = 0：红色 + 「→ 去设置 Key」

### 聊天气泡样式
- Viby 消息：左对齐，浅灰背景气泡
- 用户消息：右对齐，深色背景气泡
- 选项芯片：圆角标签，点选后转为用户气泡

### Loading 规范
- 文字生成：文字区闪烁光标
- 图片生成：浅色矩形占位 + 旋转图标
- 按钮操作中：文字变「XX中...」，禁用点击
