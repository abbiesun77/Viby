# Viby Spec V2.1

> 基于 2026-06-25 产品讨论。上一版 (V2) 的修订。

---

## 1. 核心定位

Viby 是一个**线性引导式** AI 视频前期工具。用户带一个想法或剧本进来，系统一步步带他完成：

```
想法/剧本 → 剧本确认 → 场景+分镜确认 → 资产补全 → Storyboard 生成 → 导出
```

资产管理**嵌入对话流程**，不是独立入口；工作台**作为进度看板**，让用户随时看到项目状态并恢复工作。

---

## 2. 入口：单一引导窗口

首页只有一个输入区：

```
┌────────────────────────────────────────────────┐
│  你有什么想拍的？                                │
│                                                  │
│  [  输入你的想法，或者直接粘贴剧本...        ]  │
│                                                  │
│  ○ 帮我从想法开始    ○ 我已有剧本               │
│                                                  │
│  [  开始  ]                                      │
└────────────────────────────────────────────────┘
```

---

## 3. 路径 A：只有想法（引导对话）

系统以**聊天气泡**方式逐步提问，每次只问一个问题，用户通过点击选项或自由输入回答：

```
Viby:  你想拍什么风格的视频？
       [写实纪录感]  [电影感]  [广告感]  [动画/插画感]

用户:  电影感

Viby:  大概多长？
       [15 秒]  [30 秒]  [60 秒]  [自定义...]

用户:  30 秒

Viby:  情绪基调呢？可以直接用形容词描述，比如"克制又带点荒诞"
       [   用户输入   ]
```

三问完成后系统生成剧本，进入**剧本 Tab**。

> 自答问题①：用聊天气泡而非表单——更有引导感，不像在填表格。

---

## 4. 路径 B：已有剧本

用户粘贴剧本 → 系统解析后直接进入**剧本 Tab**，不需要三问。

---

## 5. 三步工作流 Tab（顺序解锁）

```
[剧本]  →  [场景 & 分镜]  →  [Storyboard]
  ✓            (待解锁)           (待解锁)
```

每步：**系统生成 → 用户 Preview / 编辑 → 确认 → 解锁下一步**

---

## 6. Tab 1：剧本

### 展示

- 剧本全文，**可直接在线编辑**
- Viby 自动在文本里标注场景分割点

### 操作

- 自由编辑文字
- 「重新生成」整份剧本或某段
- 「确认剧本，进入场景」→ 解锁 Tab 2

---

## 7. Tab 2：场景 & 分镜

### 场景结构

每个场景是一个可展开的区块，包含若干**分镜卡**：

```
场景 01 · 便利店夜间外景
  ┌──────────────────────────────────────┐
  │ 镜 01  远景  冷光招牌，街道空旷      │
  │        主体：便利店外景              │
  │        动作：镜头缓慢推进            │
  │        氛围：孤寂、荧光蓝冷色调      │
  └──────────────────────────────────────┘
  ┌──────────────────────────────────────┐
  │ 镜 02  中景  主角推开玻璃门          │
  │ ...                                  │
  └──────────────────────────────────────┘
  [＋ 添加分镜]
```

> 自答问题②：分镜卡四个字段—— **景别/角度 · 画面主体 · 动作/内容 · 情绪/氛围**。够用，不过载。

### 资产缺口提示（内联，嵌入流程）

场景列表生成后，在「确认场景」按钮上方显示：

```
⚠️  这些角色和场景还没有参考图，建议先补齐再生成 Storyboard：

  🔴 主角（男·30岁·航天服）          [生成三视图]  [跳过]
  🔴 便利店室内                        [生成空间图]  [跳过]
  🟡 宇航头盔道具                      [生成道具图]  [跳过]
```

- 🔴 红色 = 高度影响画面一致性，强烈建议补
- 🟡 黄色 = 有则更好，可跳过

用户点击「生成」→ 系统调用图片生成，结果显示为小缩略图可预览替换。

### 操作

- 编辑分镜卡内容
- 补充或跳过资产提示
- 「确认分镜，生成 Storyboard」→ 解锁 Tab 3

---

## 8. Tab 3：Storyboard

### 宫格选项

用户在生成前选择：

```
生成格式：  [6 格]  [9 格]  [12 格]  [16 格]  [18 格]
画面比例：  ● 横版 16:9    ○ 竖版 9:16
```

> 自答问题③：默认 16 格 + 16:9。入口让用户在此处选，不在最开始就问。

### 生成逻辑

- **每个场景 = 一张 Storyboard 合成图**
- Viby 根据分镜卡 + 已有参考图，预制 GPT Image2 提示词
- 生成结果是**一整张图**，不是单格图片集合

### 操作

```
场景 01 · 便利店夜间外景
  [生成 Storyboard]

  ┌──────────────────────────────┐
  │  16 格合成图预览              │
  └──────────────────────────────┘

  [查看/编辑提示词]   [重新生成]   [下载]
```

- **只能整体重新生成**，不能单格替换（砍掉）
- 「查看/编辑提示词」允许用户微调后重新生成
- 「下载」下载该场景的 storyboard 图

---

## 9. 工作台（项目看板）

用户进入一个已有项目时，看到的是**进度看板**，而不是空白页：

```
项目：便利店宇航员

  ✅  剧本          已确认
  ✅  场景 & 分镜   4 个场景 · 16 个分镜
  ⚠️  资产          2 项缺失参考图  [去补]
  🔲  Storyboard    未生成

  [继续工作 →]

──────────────────────────────────
最近活动
  场景 02 分镜已修改
  主角三视图已生成
```

每个步骤都是可点击入口，直接跳回对应 Tab 的编辑界面。

---

## 10. 资产管理（嵌入式，非独立导航）

资产不是独立页面，而是从工作台和场景 Tab 内联访问：

- 工作台显示资产缺口摘要
- 场景 Tab 显示逐条资产提示
- 点击任何资产缩略图可预览/替换

V1 资产类型：
- 角色三视图（正面·侧面·半侧）
- 空间/场景参考图
- 关键道具图
- 风格锚点

---

## 11. 导出

工作台入口，生成一个打包文件，包含：

- 剧本文本
- 场景 & 分镜列表（结构化）
- 所有资产图
- 所有 Storyboard 图
- 每个场景的分镜提示词（供视频模型使用）

支持两种导出目标（影响提示词格式）：
- **Seedance**
- **Omni**

---

## 12. BYOK + AI 模式

- 默认：用 Viby 托管 key，消耗 Viby Credit
- 切换：用户在「设置」填入自己的 OpenAI 兼容 base URL + API key
- Credit 耗尽后引导用户切换到 BYOK
