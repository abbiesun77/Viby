# Viby Spec V2

> 基于 2026-06-25 产品讨论重写。这份文档是 V1 的设计依据，不是实现计划。

---

## 1. 核心定位（一句话）

Viby 是一个**线性引导式**的 AI 视频前期工具：用户只需要一个想法或一份剧本，系统一步步带他生成剧本 → 场景 → 分镜 → storyboard 图。

---

## 2. 入口设计

### 唯一入口窗口

首页只有一个输入区，包含两种模式的切换：

```
┌─────────────────────────────────────────────────┐
│  你有什么想做的？                                  │
│                                                   │
│  [       输入你的想法，或者粘贴剧本...        ]   │
│                                                   │
│  ○ 帮我从想法开始    ○ 我已有剧本                 │
│                                                   │
│  [  开始  ]                                       │
└─────────────────────────────────────────────────┘
```

### 路径 A：只有想法（脑暴模式）

1. 用户输入一句话
2. 系统发起一轮**三问引导**（不超过三个问题，不堆表单）：
   - 想要什么风格？（写实 / 广告感 / 电影感 / 动画感）
   - 视频大概多长？（15s / 30s / 60s / 自定义）
   - 情绪基调是什么？（开放输入或给几个选项）
3. 系统生成剧本草稿，进入**剧本 Preview 界面**

### 路径 B：已有剧本

1. 用户粘贴剧本
2. 系统直接解析，进入**剧本 Preview 界面**

---

## 3. 线性工作流（三个 Tab）

用户进入项目后，界面是一个**三步 Tab**，只能按顺序解锁：

```
[剧本] → [场景 & 分镜] → [Storyboard]
  ✓           (locked)       (locked)
```

每一步都有：**生成结果 → 用户 Preview / 编辑 → 确认 → 解锁下一步**。

---

## 4. Tab 1：剧本

### 展示内容

- 剧本全文（可直接在线编辑）
- 每个场景有简短概述（Viby 自动分场标注）

### 用户操作

- 直接编辑文字
- 重新生成某一段
- 点击「确认剧本，生成场景」→ 解锁 Tab 2

### 数据结构

```
Script
  - rawText: string          // 用户可编辑的剧本全文
  - scenes[]: {
      sceneNumber: number
      title: string
      summary: string        // Viby 自动标注的场景概述
    }
```

---

## 5. Tab 2：场景 & 分镜

### 逻辑

- 场景由 Tab 1 的剧本自动提取
- 每个场景下，Viby 生成若干个**分镜卡**（shot cards）
- 场景数量 = 后续 storyboard 的数量

### 展示内容

每个场景是一个可展开的区块：

```
场景 01 · 便利店夜间外景
  [分镜 01] 远景：便利店冷光招牌，街道空旷
  [分镜 02] 中景：主角推开玻璃门，表情疲惫
  [分镜 03] 特写：冷柜玻璃上的倒影
  
  + 添加分镜
```

每张分镜卡包含（可编辑）：
- 镜号
- 景别 / 角度
- 画面主体
- 动作描述
- 情绪 / 灯光氛围

### 素材缺口提示（内联，不单独页面）

在场景确认前，系统列出：

```
⚠️  以下角色 / 场景还没有参考图，生成 storyboard 时可能不稳定：
  · 主角（男性，30岁，穿航天服）— [现在生成参考图] [跳过]
  · 便利店室内场景 — [现在生成参考图] [跳过]
```

用户可选择现在补，也可以跳过先看 storyboard。

### 用户操作

- 编辑分镜卡内容
- 添加 / 删除分镜
- 处理（或跳过）素材缺口提示
- 点击「生成 Storyboard」→ 解锁 Tab 3

---

## 6. Tab 3：Storyboard

### 格式规则

- **每个场景 = 一张 storyboard 图**
- storyboard 使用 **4×4 = 16 格** 宫格布局
- 场景内的分镜数量决定填充多少格（分镜少则部分格留空或合并）
- 每格 = 一张 AI 生成图 + 镜号标注

### 生成流程（用户可见、可控）

```
[场景 01] · 便利店夜间外景

  [生成该场景 Storyboard]

  生成中...
  ┌───┬───┬───┬───┐
  │ 01│ 02│ 03│   │
  │   │   │   │   │
  ├───┼───┼───┼───┤
  │   │   │   │   │
  └───┴───┴───┴───┘

  [重新生成某一格]  [下载该 Storyboard]
```

### 每格 AI 生成说明

- Viby 根据分镜卡内容 + 参考图（如有）预制 GPT Image2 提示词
- 用户不需要手写提示词，但可以看到 Viby 生成的提示词并修改
- 每格可以单独重新生成

---

## 7. 提示词预制（系统内置）

Viby 包含三套预制 prompt 模板：

| 用途 | 输入 | 输出 |
|------|------|------|
| 想法→剧本 | 用户想法 + 风格/时长/情绪 | 分场剧本（有场景标注） |
| 剧本→场景&分镜 | 剧本全文 | 场景列表 + 每场分镜卡 |
| 分镜→storyboard图 | 分镜卡内容 + 参考图 | GPT Image2 提示词 |

这些模板对用户不可见，但 storyboard 生成时可以查看并编辑最终的图像提示词。

---

## 9. 待确认的问题

在实现前仍需确认：

1. **三问引导的交互形式**：是对话气泡 chat 感，还是简单的表单选项卡？
2. **分镜卡的详细程度**：V1 需要几个字段？建议最少：景别 + 主体 + 动作 + 氛围。
3. **storyboard 图的比例**：16:9 横版还是 9:16 竖版？还是根据用户在入口选择？

---

## 10. 与现有代码的关系

现有代码可以保留的：
- 认证流程（sign-in / sign-up）已基本可用
- 数据库 schema 的 projects / profiles 表结构可沿用
- 整体 Next.js 框架不变

需要重写的：
- 工作区主体界面（从 dashboard 模式改为线性 tab 模式）
- 项目创建入口（从三个按钮改为引导式对话窗口）
- Storyboard 页面（现在是空的，需要完整实现）
