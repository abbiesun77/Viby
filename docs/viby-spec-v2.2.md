# Viby Spec V2.2 — Final V1 Design Reference

> 2026-06-25 最终对齐版本。此文档冻结产品逻辑，作为实现计划的唯一依据。

---

## 1. 核心定位

Viby 是一个**线性引导式、随时可重入**的 AI 视频前期工具。

用户从想法或剧本出发，系统一步步带他完成：

```
入口 → 剧本确认 → 场景&分镜确认 → 资产补全 → Storyboard 生成 → 导出
```

每个步骤完成后都**可以随时重新进入**，通过与 AI 对话的方式进行修改。

---

## 2. 入口：两条明确路径

首页一个输入区，两个选项：

```
┌──────────────────────────────────────────────────┐
│                                                    │
│  你想拍什么？                                      │
│  [  输入你的想法或故事梗概...              ]      │
│                                                    │
│  ○ 帮我生成剧本（一句话或故事梗概都可以）          │
│  ○ 我已有完整剧本，直接进入分场                    │
│       ↑ 提示：选这个系统不会再生成剧本，           │
│         将直接解析你的剧本内容进入场景步骤。        │
│                                                    │
│  [  开始  ]                                        │
└──────────────────────────────────────────────────┘
```

### 路径 A：想法 / 故事梗概（需要生成剧本）

系统以**聊天气泡方式**逐步提问，一次只问一个：

1. 风格：`[写实纪录感]` `[电影感]` `[广告感]` `[动画/插画感]`
2. 时长：`[15秒]` `[30秒]` `[60秒]` `[自定义]`
3. 情绪基调：自由输入

三问完成 → 生成剧本 → 进入**剧本 Tab**（可编辑确认）

### 路径 B：已有完整剧本（跳过生成）

用户粘贴剧本 → 系统直接解析 → **跳过剧本 Tab，进入场景&分镜 Tab**

---

## 3. 三步工作流（顺序解锁 + 随时可重入）

```
[剧本]  →  [场景 & 分镜]  →  [Storyboard]
```

- 按顺序解锁，但解锁后可**随时点击重入**
- 重入后通过与 AI 对话修改内容
- 对话可以随时中断，已完成的内容保留在工作台

---

## 4. Tab 1：剧本

- 剧本全文可直接编辑
- AI 自动标注场景分割点
- 操作：编辑 / 局部重新生成 / 与 AI 对话调整
- 「确认剧本」→ 解锁 Tab 2

> 路径 B 用户跳过此 Tab。

---

## 5. Tab 2：场景 & 分镜

### 结构

每个场景是一个可展开区块，包含若干分镜卡。

**分镜卡字段（四项）：**
- 景别 / 角度
- 画面主体
- 动作 / 内容
- 情绪 / 氛围

```
场景 01 · 便利店夜间外景
  ┌──────────────────────────────────────┐
  │ 镜 01  远景 · 便利店外景              │
  │        动作：镜头缓慢推进             │
  │        氛围：孤寂、荧光蓝冷色调       │
  └──────────────────────────────────────┘
  [＋ 添加分镜]  [与 AI 对话调整本场景]
```

### 资产缺口提示（内联）

场景生成后，在确认按钮上方显示：

```
以下内容建议补充参考图，可提升画面一致性：

🔴 主角（男·30岁·航天服）    [生成三视图]  [跳过]
🔴 便利店室内空间             [生成空间图]  [跳过]
🟡 宇航头盔道具               [生成道具图]  [跳过]
```

- 🔴 高影响，强烈建议
- 🟡 可选，有则更好

生成结果作为资产卡片存入工作台，可随时替换。

### 操作

- 编辑分镜卡 / 添加删除分镜
- 与 AI 对话修改场景内容
- 补充或跳过资产提示
- 「确认场景，生成 Storyboard」→ 解锁 Tab 3

---

## 6. Tab 3：Storyboard

### 生成前选项

```
宫格数量：  [6格]  [9格]  [12格]  ●[16格]  [18格]
画面比例：  ●横版 16:9    ○竖版 9:16
```

### 生成规则

- **每个场景 = 一张合成 Storyboard 图**
- Viby 根据分镜卡内容 + 已有参考图预制图像提示词
- 输出是**一整张图**，不是单格集合

### 操作

```
场景 01 · 便利店夜间外景
  [生成 Storyboard]

  [合成图预览]

  [查看 / 编辑提示词]   [重新生成]   [下载]
```

- 不支持单格替换，只能**整体重新生成**
- 可编辑提示词后重新生成
- 可与 AI 对话调整后重新生成

---

## 7. 工作台（项目进度 + 资产管理入口）

### 进度看板

```
项目：便利店宇航员

  ✅  剧本          已确认             [重新编辑]
  ✅  场景 & 分镜   4场景 · 16分镜     [重新编辑]
  ⚠️  资产          2项待补充          [管理资产]
  🔲  Storyboard    未生成             [去生成]

  [导出素材包]
```

### 资产管理区

点击「管理资产」展开资产卡片列表：

```
角色
  [主角缩略图]  已生成  [替换] [重新生成] [与AI调整]
  [女配角]      ⚠️ 未生成  [生成]

场景
  [便利店室内]  已生成  [替换] [重新生成]

道具
  [宇航头盔]    ⚠️ 未生成  [生成]

风格锚点
  [风格图]      已生成  [替换]
```

每张资产卡片都可以：
- **替换**（上传本地图）
- **重新生成**（重新调用 AI）
- **与 AI 对话调整**（修改参考描述后重新生成）

---

## 8. 导出

工作台「导出素材包」按钮 → 生成一个 zip 压缩包，包含：

- `script.txt` — 剧本全文
- `scenes.json` — 场景和分镜结构化数据
- `assets/` — 所有已生成的资产图（角色/场景/道具/风格）
- `storyboards/` — 所有场景的 Storyboard 合成图
- `prompts.txt` — 每个场景的图像提示词

不区分目标平台，用户自行决定如何使用这些素材。

---

## 9. BYOK + AI 模式

- 默认：Viby 托管 key，消耗 Viby Credit
- 「设置」页：填入 OpenAI 兼容 base URL + API key 切换到 BYOK
- Credit 耗尽时引导用户切换

---

## 10. V1 明确不做

- ❌ 完整素材包导入解析
- ❌ Repair / 修复流程
- ❌ Storyboard 单格替换
- ❌ 导出平台区分（Seedance / Omni 格式）
- ❌ 复杂团队权限

---

## 11. 与现有代码的关系

**保留：** 认证流程、Supabase schema 基础表、Next.js 框架结构

**重写：**
- 项目创建入口 → 引导式聊天气泡对话
- 工作区 → 三步 Tab + 进度看板 + 资产管理
- Storyboard 页 → 宫格选择 + 合成图生成
- 资产提示 → 内联于场景 Tab + 工作台可管理
